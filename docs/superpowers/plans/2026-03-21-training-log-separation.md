# Training Log Separation — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stop Claude from destroying training program detail by separating the immutable program from append-only workout log entries.

**Architecture:** Training program stays in `system_memory` as read-only reference. New `TrainingLog` table captures workout completions, performance notes, and program adjustments as append-only entries. Claude returns `training_log_entry` (small, structured) instead of `memory_update` (full rewrite) for workout events. `memory_update` is reserved for rare full program changes only. Context assembly merges program + recent log entries so Claude sees the full picture.

**Tech Stack:** Python/FastAPI, SQLAlchemy 2.x (sync), Alembic, existing test patterns.

**Spec:** Brainstormed in conversation 2026-03-21.
